## 1. 时间窗口

- 起始：2026-04-27
- 结束：2026-07-19
- 节奏：按 2 周一个里程碑（共 6 个迭代）

## 2. 里程碑总览

| 周期 | 目标 | 关键产出 | 验收口径 |
|---|---|---|---|
| W1-W2 | 完成架构与边界收口 | Phase0 + Phase1 文档 | 目标、分层、失败策略达成共识 |
| W3-W4 | 统一抓取任务模型 | task/batch/state/checkpoint/audit | 可按批次追溯抓取过程 |
| W5-W6 | 打通 Raw->Standardized(3域) | 首批 3 表标准化规则 | 标准化可重复执行 |
| W7-W8 | 质量规则与门禁 | 规则库 + gate + 报告 | 不合格批次不可发布 |
| W9-W10 | Served 发布与服务改造 | release_version + 只读 served | 服务不依赖同步回源 |
| W11-W12 | 回放/回滚/监控/测试闭环 | runbook + tests + dashboard | 故障可定位、可恢复 |

## 3. 每阶段 DoD（完成定义）

### M1（W1-W2）
- 完成数据层/任务流/版本模型文档。
- 明确迁移计划与冻结规则。

### M2（W3-W4）
- 任务状态机可覆盖 `pending/running/succeeded/failed/partial/retrying/published`。
- Raw 审计字段完整写入。

### M3（W5-W6）
- `market_quote_daily/financial_indicator/macro_indicator` 具备统一主键与字段字典映射。

### M4（W7-W8）
- 至少 10 条关键质量规则上线并启用门禁。

### M5（W9-W10）
- 服务查询默认读 `latest_stable` 发布版本。

### M6（W11-W12）
- 关键链路具备回放测试、发布阻断测试与回滚演练记录。

## 4. 风险与缓解

- 风险：旧缓存语义残留导致边界混乱。  
  缓解：所有新增逻辑强制映射到 L0/L1/L2。
- 风险：字段映射不稳定。  
  缓解：映射版本化 + 样本快照测试。
- 风险：服务端仍有隐式回源。  
  缓解：配置开关默认关闭同步回源，并增加告警。

## 5. KPI 建议

- 发布成功率 >= 99%
- 质量门禁通过率 >= 95%
- 首批数据域 T+0 发布延迟 <= 60 分钟
- 回放成功率 >= 99%
