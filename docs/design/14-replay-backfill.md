# 重放与回补策略

> 任务编号: `T1-005`
> 状态: Draft-v1
> 最后更新: 2026-04-23

## 1. 目标

支持按批次、按日期、按分区进行数据重建与缺口回补，且不破坏既有稳定发布版本。

## 2. 回放（Replay）

### 2.1 触发方式

- 指定 `batch_id`
- 指定 `dataset + date_range`
- 指定 `dataset + partition`

### 2.2 执行步骤

1. 从 L0 读取对应原始资产
2. 重新执行 normalize + validate
3. 生成候选发布版本
4. 通过门禁后执行 publish

### 2.3 关键约束

- 回放必须带 `normalize_version`。
- 回放结果不直接覆盖线上版本，需走发布流程。

## 3. 回补（Backfill）

### 3.1 场景

- 历史窗口缺失
- 源临时故障导致断更
- 新规则上线后的历史纠偏

### 3.2 策略

- 小窗口（<= 30 天）：直接批次回补
- 中窗口（31-365 天）：按月分片回补
- 大窗口（> 365 天）：分阶段离线回补 + 发布节流

## 4. 幂等要求

- 同一回放请求生成稳定幂等键。
- 同一发布版本禁止重复覆盖。
- 回补过程中任何失败都可从 checkpoint 续跑。

## 5. 验收

- 随机抽取 3 个历史批次可成功回放。
- 断点恢复后结果与一次性执行一致。
